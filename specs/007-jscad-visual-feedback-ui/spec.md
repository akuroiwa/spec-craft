# Feature Specification: JSCAD Visual Feedback Loop and 3D Preview UI

**Feature Branch**: `007-jscad-visual-feedback`  
**Created**: 2026-04-25  
**Status**: Draft  
**Input**: User description: "Phase 7: JSCAD Visual Feedback Loop and 3D Preview UI"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Visual Feedback Loop (Priority: P1)

As a creator using an AI agent, I want the tool to generate a visual representation (SVG) of my CAD models so that the AI can "see" the result and suggest precise improvements.

**Why this priority**: This closes the loop between text-based specs and visual reality. It enables the AI to act as a true design partner by verifying dimensions and aesthetics visually.

**Independent Test**: Can be tested by triggering a CAD build and verifying that both an `.stl` and an `.svg` file are generated in the `build/cad/` directory.

**Acceptance Scenarios**:

1. **Given** a JSCAD specification, **When** I trigger the `trigger_cad_build` tool, **Then** the system uses `npx @jscad/cli` to generate both STL and SVG (projection) files.
2. **Given** a generated SVG, **When** the AI analyzes the file, **Then** it can correctly identify the proportions and features of the model.

---

### User Story 2 - Interactive 3D Preview (Priority: P2)

As a developer, I want to quickly preview my generated 3D models in a web browser so that I can verify the design before proceeding with further tactical tasks.

**Why this priority**: Human verification is essential in creative SDD. A fast, integrated previewer significantly improves the developer experience.

**Independent Test**: Can be tested by running the `spec-craft browse` command and verifying that a Three.js-based viewer opens in the browser, correctly loading the latest generated STL.

**Acceptance Scenarios**:

1. **Given** a generated STL file, **When** I execute `spec-craft browse`, **Then** a local web server starts and serves an interactive 3D preview page.
2. **Given** the preview page is open, **When** I regenerate the model, **Then** the preview reflects the changes (manually or via reload).

---

### Edge Cases

- **Missing Node/NPX**: What if the environment doesn't have Node.js installed? (Assumed: Tool provides a clear error message with installation instructions).
- **Complex Models**: What if the JSCAD model is too complex for SVG projection? (Assumed: Fallback to STL-only build with a warning).
- **Port Conflict**: What if the default preview server port is already in use? (Assumed: Tool automatically finds an available port or allows override).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: `CADDriver` MUST replace the mock implementation with actual calls to `npx @jscad/cli`.
- **FR-002**: `trigger_cad_build` MUST support an optional `formats` argument, defaulting to `['stl', 'svg']`.
- **FR-003**: System MUST provide a `spec-craft browse` CLI command that starts a lightweight Python web server.
- **FR-004**: Preview UI MUST be based on Three.js and support interactive rotation/zoom of STL files.
- **FR-005**: Preview UI MUST be bundled as a package resource (HTML/JS) within the `spec_craft` source.

### Key Entities *(include if feature involves data)*

- **CAD Model**: Source `.js` code and generated `.stl` and `.svg` artifacts.
- **Preview Server**: A local HTTP process serving static assets and build results.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Full CAD build (STL + SVG) completes in under 5 seconds for medium complexity models.
- **SC-002**: The `spec-craft browse` command starts the server and opens the browser in under 2 seconds.
- **SC-003**: 100% of generated SVG projections are valid and viewable by LLMs with vision capabilities.
- **SC-004**: The preview UI correctly renders 100% of valid STL files generated by the tool.

## Assumptions

- **Node.js**: Assumes `npx` is available in the user's PATH.
- **Vision Capabilities**: Assumes the AI agent (e.g., Gemini 1.5) supports SVG analysis.
- **Web Browser**: Assumes a modern web browser is installed on the user's system.
